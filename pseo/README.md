**Cliente:** Poncho Dávalos — Asesor Inmobiliario Certificado AMPI

## 1. Resumen Ejecutivo

Se diseñó e implementó una **Plataforma de SEO Programático (pSEO)** para el sitio web de Poncho Dávalos, transformando datos reales del mercado inmobiliario — **2,255 propiedades activas, precios promedio de $11.1M MXN, plusvalía de +10.2% y 67+ desarrollos verificados** — en un ecosistema de **55 páginas especializadas + 1 página hub central** que posicionan a Poncho Dávalos como la referencia en bienes raíces en Puerto Vallarta y Riviera Nayarit.

El sistema genera automáticamente landing pages para cada combinación de **zona geográfica × tipo de propiedad × guía editorial**, cubriendo desde "propiedades Zona Romántica Puerto Vallarta" hasta "buying property Mexico as foreigner". Cada página incluye datos reales de mercado, FAQs únicas bilingües, schema markup para Google y CTAs directos a WhatsApp.

**Diferenciador clave:** Poncho Dávalos es el **único asesor en la zona con un ecosistema de 56 páginas de inteligencia de mercado**, con datos de plusvalía por zona, ROI de renta vacacional, comparativas de inversión y guías en primera persona — contenido que ningún competidor tiene.

## 6. Pilares Tecnológicos y de Optimización

| Pilar | Implementación |
|-------|---------------|
| **Datos Estructurados (Schema.org)** | JSON-LD en cada página: `Person`, `RealEstateAgent`, `FAQPage`, `Article`, `BreadcrumbList`. Diseñado para Rich Snippets y Knowledge Graph de Google. |
| **Meta Tags Completos** | `<title>` (hasta 60 caracteres), `<meta description>` (hasta 155 caracteres), Open Graph, Twitter Cards, canonical URLs. |
| **FAQs Contextuales Bilingües** | 3-4 preguntas únicas por página en el idioma correspondiente. FAQPage schema para rich snippets en Google. |
| **Hreflang Bilingüe** | Las páginas con versión en ambos idiomas se enlazan mutuamente con etiquetas `hreflang` para que Google muestre la versión correcta según el idioma del usuario. |
| **URLs Semánticas** | Estructura `/pseo/zona-romantica-puerto-vallarta/` — descriptiva, con keyword principal en el slug. |
| **Rendimiento** | HTML estático sin JavaScript obligatorio. Se carga rápido en cualquier dispositivo. |
| **Optimización para IA** | Contenido estructurado con schema markup y FAQs que los LLMs pueden citar como fuente. |

## 8. Pipeline de Generación

### Stack Técnico

| Componente | Detalle |
|------------|---------|
| **Motor** | Node.js — integrado como Paso 8 en el pipeline de build existente |
| **Templates** | Handlebars (.hbs): `pseo-zona.hbs`, `pseo-tipo-propiedad.hbs`, `pseo-editorial.hbs`, `pseo-hub.hbs` |
| **Datos** | JSON (`pseo/data/pages.json`) — 55 registros con datos de mercado, FAQs, descripciones, proyectos |
| **Automatización** | GitHub Actions cada 48 horas — regenera páginas, sitemap y traducciones |

### Estructura de Archivos

```
pseo/
├── README.md                       # Este documento
├── ESTRATEGIA.md                   # Estrategia completa con KPIs y roadmap
├── data/
│   └── pages.json                  # 55 registros con contenido único
├── templates/
│   ├── pseo-zona.hbs               # Template para páginas de zona (Tipo A)
│   ├── pseo-tipo-propiedad.hbs     # Template para tipos de propiedad (Tipo B)
│   ├── pseo-editorial.hbs          # Template para guías editoriales (Tipo C)
│   └── pseo-hub.hbs                # Template para el hub central
├── [55 carpetas de páginas]        # Una carpeta por página con index.html
│   ├── zona-romantica-puerto-vallarta/
│   ├── condominios-venta-puerto-vallarta/
│   ├── invertir-en-puerto-vallarta/
│   └── ...
└── index.html                      # Hub central generado
```

### Ejecución

```bash
# Regenerar todas las páginas PSEO, sitemap y traducciones:
npm run build:flexmls
```

El sistema genera automáticamente:
- 55 páginas HTML (una por registro en pages.json)
- 1 página hub central (index.html)
- Entradas en el sitemap con hreflang
- Claves de traducción i18n (ES/EN)

### Escalabilidad

| Acción | Procedimiento |
|--------|---------------|
| **Agregar nueva página** | Agregar un registro en `pseo/data/pages.json` → ejecutar `npm run build:flexmls` |
| **Actualizar datos de mercado** | Editar el registro correspondiente en pages.json → regenerar |
| **Agregar nueva zona** | Agregar registro tipo A con datos de la zona → regenerar |
| **Agregar nuevo tipo de propiedad** | Agregar registro tipo B → regenerar |
| **Agregar nueva guía editorial** | Agregar registro tipo C → regenerar |
| **Tiempo estimado** | **15-30 minutos** para agregar una página nueva |
